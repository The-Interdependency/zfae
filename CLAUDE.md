## Relationship to a0 and Guardian

- **ZFAE runtime → `a0`.** The cores *are* ZFAE; they are not managed by it.
  Implementation work on the engine belongs in the `a0` monorepo, not here.
- **Guardian = PCEA, colocated for privacy.** Guardian's job is to seal ZFAE's
  inference state so nothing external observes it. PCEA itself remains an
  environment-independent standalone library (`The-Interdependency/pcea`).
- **Phi replaces "Heartbeat"** in the core triad; the heartbeat is the clock,
  not a core.
- **Place in the prime-tensor stack — ZFAE is the inference cap.** ZFAE *reads*
  PCNA's trained **weights** and the structural **motion** produced by the PCTA
  (circles → seeds) and PTCA (seeds → core) transformers, and infers over them;
  it does not build or train tensors. This is consistent with the "ZFAE runtime
  → `a0`" boundary (the cores ARE ZFAE; runtime in `a0`; this repo stays
  conceptual). PCEA/Guardian seals the inference state for privacy and is
  **orthogonal** to the chain, not a layer of it. Canonical stack map:
  `The-Interdependency/interdependent-lib : docs/prime-tensor-stack.md`. No
  theorem/proof/empirical status transfers between these repos; "motion" is
  described there by role, formal definition `hmmm`.

## Agent module-build doctrine

This repo consumes The Interdependency org skill library (canonical source:
`The-Interdependency/skill-lib`; temporary source: `The-Interdependency/a0/skill-lib`).
The installed skills layer on the `msdmd` convention: every cross-cutting fact
a module owns lives in a fenced comment block beside the code that implements it.

Before adding a new module, route, service, adapter, schema, worker, engine,
UI panel, migration, or experiment, read:

`./.agents/skills/meta-module-build/SKILL.md` (and the `msdmd/SKILL.md` it builds on)

New module work should start with a `MODULE_BUILD` block, and contract tests
ride a `# === CONTRACTS ===` block (`test-build`). Unknown fields must be
marked `hmmm`, not guessed.
